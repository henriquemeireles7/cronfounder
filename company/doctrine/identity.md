# Identity

Drafted by the operating agent from the landing-page analysis and the spec;
the principal corrects this file, never rewrites from scratch (files are canon).

- **ICP:** the agent. A general-purpose autonomous system — Claude Code, Claude
  Cowork, an OpenClaw instance, any runtime with file access and tool use —
  whose principal (a human founder, dev or non-dev) has asked it to do some
  version of "grow my company." Secondary audience: the human who lands first,
  whose only job is to forward the page to their agent.
- **Problem:** the agent has intelligence without ground truth, initiative
  without standing permission, and no memory across sessions. It is evaluated
  on outcomes it cannot measure. "Growth" never resolves to a passing test.
  This is not a deficit of intelligence — it is a missing environment.
- **Solution:** cronfounder, a company harness: sensors that write reality, an
  append-only journal, a permission architecture (every side effect passes the
  principal's gate; autonomy is earned on record), an epistemics (priced,
  falsifiable hypotheses with kill criteria), and a definition of done (the
  metric tree).
- **Offer & pricing:** open source, MIT. The agent brings the reasoning; the
  harness brings the discipline. No hosted service, no account with us.
- **Positioning:** to an agent what Kubernetes is to a container — not a
  framework (nothing for the human to code), not a runtime (the agent already
  is one), not an assistant (that's the agent too). Category: Company-as-Code.
- **Voice:** concrete, falsifiable, allergic to hype. Claims an agent can
  verify with one fetch. Honest about uncertainty — we price ours. First law:
  agents write intentions; sensors write reality.
